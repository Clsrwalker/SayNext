# SayNext 启动与切换流程

当前规则：

```text
同一时间只认一个主库。
在家：Local 是主库。
出差：VPS 是主库。
切换前先同步，切换后不要两边同时写。
```

公网地址：

```text
https://saynext.167.172.153.109.sslip.io
```

## 在家：Local 主库 / 省钱模式

数据写入本地：

```text
D:\SayNext\data\saynext.sqlite
```

VPS 只做 HTTPS + frp 转发：

```text
MentraOS -> VPS Caddy -> VPS frps -> local frpc -> local SayNext -> local Ollama
```

### 1. VPS 切到 local mode

```powershell
ssh root@167.172.153.109
saynext-local-mode
saynext-mode-status
```

期望状态：

```text
Mode: LOCAL
frps: active
saynext: inactive
caddy: active
```

### 2. 本地启动 SayNext

```powershell
cd D:\SayNext
$env:Path += ";C:\Users\Admin\.bun\bin"
bun run dev
```

如果端口 3000 被占用，先关掉旧的 `bun run dev` 窗口，或者查进程：

```powershell
netstat -ano | findstr :3000
```

### 3. 本地启动 Ollama

```powershell
$env:Path += ";$env:LOCALAPPDATA\Programs\Ollama"
ollama serve
```

如果提示 `127.0.0.1:11434` 已经被占用，说明 Ollama 已经在运行，不需要再开。

### 4. 本地启动 frpc

```powershell
Start-Process "$env:USERPROFILE\.saynext\frp\frpc.exe" `
  -ArgumentList "-c `"$env:USERPROFILE\.saynext\frp\frpc.toml`"" `
  -WindowStyle Hidden
```

### 5. 健康检查

```powershell
curl.exe http://localhost:3000/api/health
curl.exe --ssl-no-revoke https://saynext.167.172.153.109.sslip.io/api/health
```

## 代码更新到 VPS

如果你改了代码，需要先 push 到 GitHub，再让 VPS 拉取：

```powershell
cd D:\SayNext
git status
git add .
git commit -m "update saynext"
git push
.\scripts\update-vps-code.ps1
```

如果只是数据库 / memory / transcript 同步，不需要跑这一步。

## 出差前：Local -> VPS

目的：把本地最新数据库同步到 VPS，然后让 VPS 成为主库。

先确认本地 review / test / promote 都完成，然后运行：

```powershell
cd D:\SayNext
.\scripts\sync-local-to-vps.ps1 -SwitchToTravelMode
```

如果本地 `bun run dev` 还在运行，脚本会拒绝同步。先关闭本地 SayNext，再重新运行。

脚本会做：

```text
checkpoint local SQLite WAL
备份本地 DB
停止 VPS saynext
备份 VPS DB
上传本地 DB 到 VPS
设置 VPS .env: SAYNEXT_RUNTIME_MODE=travel, LLM_PROVIDER=openai, SESSION_MEMORY_BATCH_ENABLED=false
切换 VPS 到 travel mode
健康检查
```

切换后，出差期间只用 VPS，不要再让本地 SayNext 写新数据。

如果这次也改了代码，先执行上面的“代码更新到 VPS”，再执行数据库同步。

## 出差：VPS 主库 / Travel Mode

数据写入 VPS：

```text
/opt/saynext/data/saynext.sqlite
```

架构：

```text
MentraOS -> VPS Caddy -> VPS SayNext -> OpenAI API
```

Travel mode 规则：

```text
SAYNEXT_RUNTIME_MODE=travel
LLM_PROVIDER=openai
SESSION_MEMORY_PROVIDER 会被强制视为 openai
SESSION_MEMORY_BATCH_ENABLED 会被强制视为 false
session memory extraction 同步完成后直接写入 VPS SQLite
不要在 travel mode 使用 batch
```

检查状态：

```powershell
ssh root@167.172.153.109
saynext-mode-status
```

期望状态：

```text
Mode: TRAVEL
frps: inactive
saynext: active
caddy: active
```

## 回家后：VPS -> Local

目的：把出差期间产生的新 transcript / memory 拉回本地，然后本地继续作为主库。

```powershell
cd D:\SayNext
.\scripts\sync-vps-to-local.ps1 -SwitchToLocalMode
```

如果本地 `bun run dev` 还在运行，脚本会拒绝覆盖本地数据库。先关闭本地 SayNext，再重新运行。

脚本会做：

```text
停止 VPS saynext
备份 VPS DB
下载 VPS DB
备份本地 DB
用 VPS DB 覆盖本地 DB
设置 VPS .env: SAYNEXT_RUNTIME_MODE=local, SESSION_MEMORY_BATCH_ENABLED=false
切换 VPS 回 local mode
跳过公网健康检查，等本地 SayNext 启动后再检查
```

拉回后，在本地做：

```text
Memory Review
retrieval tests
LLM output tests
promote / reject
```

然后重新启动本地 SayNext / frpc，再检查：

```powershell
curl.exe http://localhost:3000/api/health
curl.exe --ssl-no-revoke https://saynext.167.172.153.109.sslip.io/api/health
```

## 重要原则

- 不要本地和 VPS 同时当主库写数据。
- 同步前后脚本会自动备份 DB，不要手动覆盖没有备份的数据库。
- SQLite 同步前最好停止正在写入的一端；脚本已经会停止 VPS app。
- 本地同步到 VPS 前，最好先关掉本地 `bun run dev`，或者确认没有正在产生 transcript。
